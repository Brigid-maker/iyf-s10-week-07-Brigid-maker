ESLint & Prettier

Task 14.4 — Linting · Formatting · Code Quality Setup

ESLint

A linter — it analyses your code for logical errors, bad practices, and rule violations. It reads your code and flags problems like using var, unused variables, or missing curly braces. It does not touch formatting.

Prettier

A formatter — it automatically rewrites your code's layout: indentation, quotes, semicolons, line length. It does not check for logic or bad practices. It only cares about how your code looks.

1

Initialise npm Project

Creates a package.json file — the project's configuration file. The -y flag accepts all defaults automatically.

terminal 
npm init -y
2

Install ESLint

Installs ESLint as a dev dependency (not needed in production). Then run --init to answer setup questions and auto-generate your config file.

terminal 
npm install eslint --save-dev
npx eslint --init
3

Configure ESLint

Create .eslintrc.json in your project root. Each rule can be set to "error" (blocks), "warn" (flags), or "off".

.eslintrc.json 
{
  "env": {
    "browser": true,     // enables window, document, etc.
    "es2021":  true      // enables modern JS syntax
  },
  "extends": "eslint:recommended",  // sensible rule defaults
  "parserOptions": {
    "ecmaVersion": "latest",
    "sourceType":  "module"   // enables import/export
  },
  "rules": {
    "no-unused-vars": "warn",   // yellow warning for unused vars
    "no-console":     "off",    // allow console.log freely
    "eqeqeq":         "error",  // force === instead of ==
    "curly":          "error",  // always use { } in if/for
    "prefer-const":   "warn"    // prefer const over let when possible
  }
}
Rule Level What it catches
no-unused-vars warn Variables you declared but never used — usually a mistake
no-console off Disabled — so you can use console.log freely while learning
eqeqeq error Forces ===. Using == causes subtle type-coercion bugs
curly error Requires curly braces on all if/for blocks — prevents off-by-indent bugs
prefer-const warn Suggests const when a variable is never reassigned
4

Install & Configure Prettier

Prettier handles all formatting automatically — indentation, quotes, semicolons, trailing commas. Configure it with .prettierrc.

terminal 
npm install prettier --save-dev
.prettierrc 
{
  "semi":          true,    // add semicolons at end of lines
  "singleQuote":  true,    // use 'single' instead of "double" quotes
  "tabWidth":      4,       // indent with 4 spaces
  "trailingComma":"es5",   // trailing commas where ES5 allows
  "printWidth":   80       // wrap lines at 80 characters
}
5

Add Scripts to package.json

These scripts let you run linting and formatting with short commands from the terminal instead of typing long paths each time.

package.json — scripts section 
{
  "scripts": {
    "lint":       "eslint src/",           // check for errors
    "lint:fix":   "eslint src/ --fix",     // auto-fix what it can
    "format":     "prettier --write src/"    // reformat all files
  }
}
terminal — how to use them 
npm run lint       # shows all ESLint errors/warnings
npm run lint:fix   # auto-fixes safe issues (like prefer-const)
npm run format     # rewrites formatting in all src/ files
6

Install VS Code Extensions

These extensions run ESLint and Prettier live in your editor as you type, without needing to run terminal commands manually.

🔍
ESLint Underlines problems in red/yellow as you type. Shows rule name on hover.
✨
Prettier Auto-formats your file every time you hit Save (Ctrl+S).
VS Code settings.json — enable Format on Save 
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode"
}
