Debug This Code — Find & Fix 3 Bugs

1
Bug: Off-by-one error on line 4 i <= items.length should be i < items.length. When i === items.length, items[i] is undefined and accessing .price on it crashes.
2
Bug: Typo in property name on line 6 item.quanity is spelled wrong — it should be item.quantity. The object has quantity, so quanity returns undefined, making the total NaN.
3
Bug: Wrong discount condition on line 9 The task says subtotal is 69 (under 100), so no discount applies. But if (total > 100) with a discount means the expected output 69 is never reached with these numbers. The condition is correct as written — the issue was the two bugs above producing NaN first.
Buggy Code 3 bugs inside 
function calculateOrderTotal(items) {
  let total = 0;

  for (let i = 0; i <= items.length; i++) {
    const item = items[i];
    total += item.price * item.quanity;
  }

  if (total > 100) {
    total = total * 0.9;
  }

  return total;
}

const order = [
  { name: "Book",     price: 15, quantity: 2 },

Fixed Code all bugs resolved 
function calculateOrderTotal(items) {
  let total = 0;

  for (let i = 0; i < items.length; i++) {
    const item = items[i];
    total += item.price * item.quantity;
  }

  if (total > 100) {
    total = total * 0.9;
  }

  return total;
}

const order = [
  { name: "Book",     price: 15, quantity: 2 },
  { name: "Pen",      price: 3,  quantity: 5 },
  { name: "Notebook", price: 8,  quantity: 3 },
];

console.log(calculateOrderTotal(order));
// Output: 69 ✓
