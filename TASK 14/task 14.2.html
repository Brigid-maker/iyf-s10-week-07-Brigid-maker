Task 14.2 — Clean Code Practices

Three exercises: Meaningful Names · Single Responsibility · No Magic Numbers

🏷️

Meaningful Names

Names should reveal intent. A reader should know what a variable does without reading the surrounding code.

🎯

Single Responsibility

Every function should do exactly one thing. If you can describe it with "and", split it up.

🔢

No Magic Numbers

Replace raw numbers and strings with named constants. The name explains why that value exists.

Meaningful Names

Rename variables and functions so the code explains itself

Before Bad 
// What does any of this mean?
const d = new Date();

const x = users.filter(
  u => u.a > 18
);

function calc(a, b) {
  return a * b * 0.1;
}

let f = false;
function chk(p) {
  if (p.l > 8) f = true;
  return f;
}
After Good 
// Intent is crystal clear
const currentDate = new Date();

const adultUsers = users.filter(
  user => user.age > 18
);

function calculateDiscount(
  price, quantity
) {
  const DISCOUNT_RATE = 0.1;
  return price * quantity * DISCOUNT_RATE;
}

let isPasswordValid = false;
function validatePassword(password) {
  isPasswordValid = password.length > 8;
  return isPasswordValid;
}

Single Responsibility

Split a function that does 5 things into 5 focused functions

Before — processUser() does everything Bad 
function processUser(userData) {
  // 1. Validate
  if (!userData.email.includes("@"))
    throw new Error("Invalid email");
  if (userData.age < 18)
    throw new Error("Must be adult");

  // 2. Transform
  userData.email =
    userData.email.toLowerCase();
  userData.name =
    userData.name.trim();

  // 3. Save  4. Email  5. Update UI
  database.save(userData);
  emailService.send(userData.email);
  document.getElementById("status")
    .textContent = "User created!";
}
After — one job per function Good 
function validateUser(userData) {
  if (!userData.email.includes("@"))
    throw new Error("Invalid email");
  if (userData.age < 18)
    throw new Error("Must be adult");
}

function normaliseUser(userData) {
  return { ...userData,
    email: userData.email.toLowerCase(),
    name:  userData.name.trim(),
  };
}

async function createUser(userData) {
  validateUser(userData);
  const clean = normaliseUser(userData);
  await database.save(clean);
  await emailService.sendWelcome(
    clean.email
  );
  return clean;
}

No Magic Numbers

Replace bare literals with named constants that explain their purpose

Before — numbers with no context Bad 
if (password.length < 8) { }

setTimeout(callback, 86400000);

if (response.status === 404) { }

if (score >= 90) return "A";
if (score >= 75) return "B";
if (score >= 60) return "C";

const tax = price * 0.16;
const discounted = price * 0.85;
After — named constants explain the why Good 
const MIN_PASSWORD_LENGTH = 8;
if (password.length < MIN_PASSWORD_LENGTH) {}

const ONE_DAY_MS = 24 * 60 * 60 * 1000;
setTimeout(callback, ONE_DAY_MS);

const HTTP_NOT_FOUND = 404;
if (response.status === HTTP_NOT_FOUND) {}

const GRADE_A_MIN = 90, GRADE_B_MIN = 75,
      GRADE_C_MIN = 60;
if (score >= GRADE_A_MIN) return "A";

const VAT_RATE      = 0.16;
const DISCOUNT_RATE = 0.15;
const tax        = price * VAT_RATE;
const discounted = price * (1 - DISCOUNT_RATE);

Quick Reference Checklist

Run through this before every code review

🏷️ Variable names reveal intent without comments
🎯 Each function does exactly one thing
🔢 No bare numbers or strings — use constants
📏 Functions are under ~20 lines
🚫 No variables named d, x, tmp, data
🔁 No duplicate code — extract shared logic
⚠️ No more than 3 levels of nesting
✅ const by default, let only when needed
